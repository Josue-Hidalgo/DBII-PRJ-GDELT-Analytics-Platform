Semestre: Mayo 2026

## Objetivo General

Desarrollar un pipeline de análisis de datos basado en eventos mundiales utilizando información del proyecto GDELT. La solución debe ejecutarse completamente en contenedores y estar compuesta por tres etapas principales:

1. Carga de datos (Loader)
2. Procesamiento y análisis con Apache Spark
3. Visualización de resultados almacenados en MongoDB

Los resultados finales deben presentarse mediante una página web o dashboard.

---

# Requisitos de Arquitectura

La solución debe ejecutarse utilizando contenedores Docker.

Como mínimo se requieren:

* 1 contenedor Loader
* 2 o más contenedores Spark
* 1 contenedor MongoDB
* 1 contenedor para la capa de visualización

Herramientas sugeridas:

* Docker / Docker Compose
* Apache Spark
* Apache Airflow (opcional)
* Apache Parquet
* HDFS (opcional)
* MongoDB
* Apache Superset (opcional)

La arquitectura exacta puede variar siempre que cumpla con los requisitos funcionales.

---

# Fuente de Datos

Sitio oficial:

https://www.gdeltproject.org/

Se deben descargar únicamente los archivos generados cada 15 minutos.

Archivos permitidos:

* Events Table
* Mentions Table
* GKG (Global Knowledge Graph)

Importante:

* NO descargar el histórico completo.
* El histórico completo supera los 2.5 TB.
* Solo se deben descargar ventanas pequeñas de datos recientes.

---

# Loader

## Responsabilidades

El Loader debe:

1. Ejecutarse automáticamente cada 15 minutos.
2. Descargar los archivos GDELT recientes.
3. Transformar los datos si es necesario.
4. Guardarlos en:

   * Apache Parquet
   * HDFS
   * o ambos

## Consideraciones

* Los datos RAW son muy pesados.
* Se recomienda conservar únicamente entre 1 hora y 24 horas de datos crudos.
* Los datos viejos pueden eliminarse después de ser procesados.

---

# Procesamiento con Spark

Los análisis deben ejecutarse utilizando Apache Spark leyendo desde:

* HDFS
* Parquet

Los resultados NO deben calcularse en tiempo real desde el dashboard.

Todos los resultados deben ser calculados previamente y almacenados en MongoDB.
